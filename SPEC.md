# Unfold · 电子手帐 — 技术方案

## 1. 产品定位与愿景

一个帮用户在情绪模糊状态下看见自己内心的网页日记工具。核心体验是"内心状态视觉化"——不是写完日记再配图，而是用拼贴画布作为情绪的入口。

日系简洁可爱插画风。兔子为圆润黑白线条风格，不主动打扰用户，在关键时刻温柔出现。

---

## 2. 技术选型

| 层级 | 选型 | 理由 |
|---|---|---|
| 核心框架 | 原生 HTML + CSS + JS（单文件） | 快速迭代，无需构建工具 |
| 画布引擎 | **Fabric.js 6.x**（CDN） | 对象管理、序列化、拖拽旋转缩放、图层全支持，文档完善 |
| AI 贴纸 | Anthropic Claude + 图片模型 | 现有 API Key 可复用 |
| 存储 | IndexedDB（本地） | 大文件（画布图片）存储，容量大 |
| 部署 | Vercel（后续迁移） | — |

## 3. 功能模块

### 3.1 月历首页（Month View）
- 日系插画风月历，显示当月所有日期
- 有内容的日期格显示**贴纸缩略图**（画布快照）
- 点击日期格进入当日画布编辑器
- 日期格左上角显示天气心情图标
- 今日入口居中突出显示

### 3.2 画布编辑器（Canvas Editor）
- **画布尺寸**：竖版 A5（宽 640 × 高 900 px），支持缩放适应屏幕
- **背景**：可选纯色 / 渐变 / 纸质纹理（米白格纹 / 点阵 / 水彩）
- **对象类型**：
  - 文字块（可调字体/大小/颜色/对齐，带圆角背景）
  - 图片（拖入或点击上传，自动适应画布宽度）
  - 预设贴纸（天气心情 / 装饰 / 手绘风图标，约 50 个 SVG 贴纸）
  - AI 生成贴纸（输入关键词 → 调用 AI 画图 → 加入画布）
  - 天气心情选择器（太阳 / 雨天 / 雪天 / 开心 / 难过 / 愤怒 / 平静等，10 种）
- **交互**：自由拖拽 / 旋转 / 缩放 / 改变层级
- **操作**：撤销（Ctrl+Z）/ 重做 / 删除选中 / 清空画布
- **保存**：自动保存到 IndexedDB；也可手动保存

### 3.3 精灵兔子（Bunny）
- 右下角悬浮，默认收起为一个小头像（黑白线条兔子）
- **不主动打扰用户**，不弹窗，不发通知
- 点击兔子后展开对话气泡，支持文字输入
- 兔子拥有**叙事疗法 + 艺术疗愈**背景，说话温柔简短，帮用户把模糊感受变清晰
- **触发时机**：胶囊拆封时自动出现；用户主动点击时出现

### 3.4 时间胶囊系统（Time Capsule）
- 在画布编辑器内点击"封存时刻"按钮触发
- 用户可选填：
  - 一段**写给未来自己的话**
  - 一个希望兔子在未来**提问的问题**
  - 拆封时间（1个月 / 3个月 / 6个月 / 1年 / 自定义日期）
- 封存后：当日日期格出现**小信封图标**标记
- **拆封流程**：
  1. 用户回到月历，发现信封闪烁
  2. 点击后兔子轻声提示："有东西在等你"
  3. 先呈现用户写给自己的话
  4. 兔子提出被封存的问题
  5. 画布展开，用户当下回应
  6. 可选择再封存一个新胶囊

## 4. 信息架构

```
月历首页
  └── [今日] 画布编辑器
        ├── 工具栏（文字 / 图片 / 贴纸 / AI生成 / 天气心情）
        ├── 画布区域（Fabric.js）
        ├── 兔子对话浮层
        └── 封存时刻浮层
  └── [历史日期] 画布编辑器（只读模式，可复制到今日继续编辑）
```

---

## 5. 数据模型

### CanvasEntry（日记画布）
```js
{
  id: string,              // UUID
  date: string,           // "2026-04-07"
  canvasData: object,     // Fabric.js JSON（对象结构+位置+样式）
  weather: string,        // "sunny" | "rainy" | ...
  mood: string,           // "happy" | "sad" | ...
  capsule: Capsule | null,
  createdAt: timestamp,
  updatedAt: timestamp
}
```

### Capsule（时间胶囊）
```js
{
  id: string,
  entryId: string,
  message: string,        // 写给未来自己的话
  question: string,       // 封存的问题（兔子会在拆封时问）
  openDate: string,       // "2026-07-07"
  opened: boolean,
  openedAt: timestamp | null
}
```

## 7. 实现优先级

| 阶段 | 内容 | 产出 |
|---|---|---|
| **Phase 0** | 技术方案 + 基础框架 | 本文档 + 可运行的空画布 |

| Phase 7 | 数据持久化（IndexedDB）+ 画布快照 |

## 8. 设计规范

### 色彩系统
```
--bg:        #faf8f5   // 米白背景
--surface:   #ffffff
--ink:       #2c2825   // 深棕墨水色
--ink2:      #8a7f75   // 次要文字
--ink3:      #c4bab0   // 占位文字
--accent:    #e07a5f   // 珊瑚粉（强调）
--accent2:   #81b29a   // 薄荷绿（次要强调）
--warn:      #f2cc8f   // 暖黄（提醒）
--border:    #e8e2d9
```

### 字体
- 标题/品牌：`Noto Serif SC`（衬线，典雅）
- 正文/UI：`Noto Sans SC`（无衬线，简洁）
- 画布手写风文字：`ZCOOL XiaoWei`（站酷小薇）

### 兔子设计
- 圆润黑白线条风格
- 状态：收起（小头像）/ 展开（对话中）/ 封存提示（信封在嘴边）
- 线条粗细：2px，圆润转角
